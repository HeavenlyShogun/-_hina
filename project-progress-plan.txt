## 目前技術基底

- 前端：React 19 + Vite + Tailwind CSS
- 狀態分層：`App.jsx` + custom hooks + `AudioConfigContext` + `PlaybackContext`
- 音訊：Web Audio API，自製 `audioEngine.js`
- 播放：自製 `playbackController.js`，採 absolute audio time + lookahead scheduling
- 雲端：Firebase Auth + Firestore
- 樂譜來源：同時支援舊文字譜與 JSON score

## 已完成

### 1. 播放與演奏核心

- 已完成 `audioEngine.js`，具備 `AudioContext` 初始化、compressor、reverb bus、noise buffer、voice lifecycle 管理。
- 已建立 tone preset 架構，現有音色包含：
  - `piano`
  - `lyre-long`
  - `lyre-short`
  - `flute`
  - `tongue-drum`
  - `classic`
- 已支援 live note 與 scheduled note 兩條路徑。
- 已加入 voice limit 與 voice stealing，避免過多重疊音導致失控。
- 已可依 tone preset 套用 ADSR、filter、noise、release 行為。

### 2. 精準播放排程

- `playbackController.js` 已改為以 `audioContext.currentTime` 為準的播放模型。
- `setTimeout / setInterval` 不再直接當節拍器，而是作為 scheduler 巡檢機制。
- 已拆分：
  - 音訊排程迴圈
  - UI 視覺更新迴圈
- 已支援：
  - play
  - pause
  - resume
  - stop
  - seek
  - playback rate 調整
- 已支援進度列同步與鍵位高亮同步。

### 3. 樂譜格式與資料正規化

- 已建立 `scoreDocument.js`，統一管理：
  - `title`
  - `rawText`
  - `sourceType`
  - `compiledEvents`
  - `bpm`
  - `timeSigNum`
  - `timeSigDen`
  - `charResolution`
  - `globalKeyOffset`
  - `accidentals`
  - `scaleMode`
  - `tone`
  - `reverb`
- 已支援兩種樂譜來源：
  - 舊文字譜
  - JSON score
- `score.js` 已可將文字譜或 JSON score 正規化為統一播放事件。
- JSON score 已支援 track 與 chord event 基本模型。

### 4. React 架構分層

- `App.jsx` 仍是高層整合點，但已不是單體塞滿邏輯。
- 已將主要責任拆到 hooks：
  - `useScoreState`
  - `useScorePlayback`
  - `useCloudScores`
  - `useAudioEngine`
- 已加入 context：
  - `AudioConfigContext`
  - `PlaybackContext`
- `PianoRoom.jsx` 目前維持容器層角色，負責組裝：
  - `AppHeader`
  - `PianoKeys`
  - `InstrumentSelector`
  - `ControlPanel`

### 5. UI 與互動

- 已有可用的虛擬琴鍵互動與鍵盤映射。
- 已有 `InstrumentSelector`，音色切換走 context/state，同步到音訊與譜面設定。
- `ControlPanel` 已可控制：
  - BPM
  - 拍號
  - grid / char resolution
  - 音量
  - reverb
  - global key offset
  - scale mode
- 已有 toast 提示、播放進度條、背景動態粒子與整體視覺主題。

### 6. 雲端曲庫

- 已完成 Firebase 匿名登入連線流程。
- 已完成 Firestore 曲庫訂閱與載入。
- 已支援：
  - 儲存單首樂譜
  - 刪除樂譜
  - 清空曲庫
  - 本機多檔匯入後上傳雲端
- 已能處理 legacy record 與新 `scoreDocument` 的相容正規化。

### 7. 匯入匯出與示範資料

- 已支援本機匯入：
  - 單一文字譜 / JSON 載入到編輯器
  - 多檔案批次上傳雲端
- 已支援本機匯出目前樂譜。
- 已內建 JSON demo 載入流程。

## 目前進行中

### A. 架構穩定後的第二階段整理

- 專案已從「快速做出可用功能」進入「統一資料流與整理 UI」階段。

- 文件內容與實際程式碼已有明顯落差，現在需要同步補齊 README、交接說明與 AI 用摘要，尤其是 practice mode、playhead、converter、MIDI import。

### B. 樂譜模型升級的中繼期

- 現在已有 JSON score 基礎，但還沒有完整擴充到成熟 DAW 式模型。
- converter 與 MIDI import 已經開始產出 V2 JSON，下一步應該收斂這份 schema，而不是再讓格式自由擴散。
- 多 track 是「資料格式已留入口」，但不是完整編輯工作流。
- chord 已支援基本展開與 strum，articulation / dynamics / section metadata 仍未正式完成。

### C. 播放系統進階能力

- pause / resume / seek 已存在，但還沒做 section loop、A/B loop、段落播放等完整控制。
- 現在已經有播放頭時軸與 click-to-seek 視覺入口，代表段落播放與 section control 的 UI 基礎已經出現。
- playback rate 邏輯已存在，但 UI 尚未完整暴露成使用者可直覺操作的功能。
- 長曲、複雜曲目的壓力測試與邊界驗證仍值得補。

### D. UI/UX 細修

- `InstrumentSelector` 功能已可用，但視覺表現仍偏簡單，尚未完全達到主題化介面水準。
- `SheetDisplay` 已新增練習回饋與統計面板，但仍需要實際操作驗證資訊密度、提示節奏與手機版可用性。
- 手機版與小螢幕體驗還需要進一步檢查按鍵密度、控制面板層級與可點擊區域。
- 樂譜編輯、曲庫、播放狀態之間的提示仍可更清楚。

## 建議暫時不要做的事

- 不要重寫整個 `App.jsx`。
- 不要把播放邏輯退回單純 `setTimeout`/`setInterval` 直接發聲。
- 不要把音訊實作塞回 UI component。
- 不要在還沒定義清楚 score schema 前先做大量複雜編輯 UI。
- 不要直接覆蓋現有資料流：`App/hooks/context -> PianoRoom -> presentation components`。
