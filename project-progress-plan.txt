目前音色設計方向

1. lyre-long
定位：
- 柔和
- 有尾韻
- 適合旋律線條

做法：
- triangle
- 很短 attack
- 快速 decay
- 低 sustain
- 中短 release
- 少量瞬態 noise 模擬撥弦

2. lyre-short
定位：
- 更乾淨
- 更短促
- 更清脆

做法：
- triangle
- 更短 ADSR
- 更少延音

3. flute
定位：
- 平滑
- 氣流感
- 長音表現較自然

做法：
- sine
- 較長 attack
- 較高 sustain
- 較平順 release
- 很少量 noise 模擬吹氣

4. tongue-drum
定位：
- 空靈
- 有打擊感
- 有共鳴尾音

做法：
- triangle
- 中短 attack
- 較長 decay / release
- 偏低頻濾波收尾
- 保留敲擊瞬態

5. piano
定位：
- 明亮
- 顆粒感
- 作為通用預設音色

目前狀態管理方式

目前專案狀態是集中式但不混亂的：

- App.jsx 作為頂層狀態持有者
- 透過 hook 管理：
  - 樂譜狀態
  - 雲端狀態
  - 播放狀態
  - 音訊設定
- PianoRoom.jsx 負責把狀態與事件往下傳
- 子元件盡量維持展示層 / 輕互動層角色

這代表未來擴充時應遵守：

- 不把音訊引擎實作塞進 UI component
- 不把畫面排版責任塞回 App.jsx
- 讓新音色、新控制器、新視覺模組都能沿著同一條資料流擴充

未來規劃

1. 音色系統繼續擴充
可能方向：
- 加入更多 tone preset，例如：
  - harp
  - bell
  - strings
  - pad
  - plucked synth
- 將 preset 抽成獨立設定檔
- 支援更細的 tone params 調整 UI

2. 更真實的聲學表現
可能方向：
- 雙 oscillator 疊層
- detune 微小失諧
- velocity 對濾波 / 音量 / attack 的影響更明顯
- 更精細的 reverb / delay 設計
- 不同樂器用不同 resonance/filter 模型

3. 播放系統進一步增強
可能方向：
- pause / resume 更細緻的相位控制
- seek / loop / section playback
- 更穩定的長曲播放
- 進階播放事件同步（音符高亮、進度軸、段落定位）

4. UI/UX 優化
可能方向：
- 把 InstrumentSelector 做得更視覺化
- 改善手機版排版
- 更強的演奏回饋動畫
- 更清楚的播放狀態與樂譜狀態提示
- 將舊控制 UI 整理成更一致的設計系統

5. 樂譜能力擴充
可能方向：
- 更完整的 JSON score 格式
- 多 track
- chord / articulation / dynamics
- 匯入匯出更穩定
- 範例曲庫與預設模板

6. Firebase / 雲端體驗優化
可能方向：
- 樂譜分類
- 標籤 / 搜尋
- 自動儲存
- 匿名登入以外的使用者身份機制
- 更完整的同步提示

--------------------------------------------------

目前希望外部 AI 幫忙的方向

如果要讓另一個 AI 協助，最適合的任務是：

- 設計新的 tone preset
- 協助調校 ADSR / filter / noise 參數
- 優化 React component 結構
- 提出 ControlPanel 與 InstrumentSelector 的整合方案
- 規劃樂譜格式與播放事件模型
- 改善行動端 UI/UX

不希望它做的事：

- 一次重寫整個 App.jsx
- 直接覆蓋大量檔案而不考慮現有資料流
