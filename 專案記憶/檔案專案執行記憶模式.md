# 檔案專案執行記憶模式

更新日期：2026-05-08

本檔集中保存之後處理 Project Hina 檔案與專案執行時要遵守的記憶模式。除非使用者另有指示，後續改檔、測試、整理記憶時以本檔為準。

## 記憶資料夾

| 類型 | 路徑 | 用途 |
|---|---|---|
| 專案記憶總資料夾 | `專案記憶/` | 集中放版本記憶、執行規則、後續交接摘要。 |
| 鋼琴邏輯版本記憶 | `專案記憶/鋼琴彈奏邏輯版本記憶.md` | 保存 1/2/3 版彈奏邏輯定義。 |
| 專案執行記憶模式 | `專案記憶/檔案專案執行記憶模式.md` | 保存本檔規則。 |
| 既有進度記憶 | `project-hina-progress-memory.md` | 保留既有進度紀錄，可追加摘要入口。 |

## 檔案處理規則

| 規則 | 記憶定義 |
|---|---|
| 編碼 | `src/`、`風物之琴譜/`、`樂理底層邏輯/`、`專案記憶/` 優先維持 UTF-8。 |
| 先讀再改 | 改任何檔案前先讀相關檔案，避免覆蓋使用者既有變更。 |
| 不回退他人變更 | 不使用 `git reset --hard` 或還原未確認的使用者變更。 |
| 小範圍修改 | 只改與任務直接相關的檔案；不要順手大改架構。 |
| 記憶集中 | 新增長期規格、版本表、交接摘要時，優先放在 `專案記憶/`。 |
| 舊檔保留 | 舊版邏輯與舊譜相容資料不刪除，改以版本標籤標記。 |

## 專案執行規則

| 情境 | 執行方式 |
|---|---|
| 找檔案 / 找文字 | 優先使用 `rg`、`rg --files`。 |
| 檢查編碼 | 使用 `node scripts/ensure-utf8.mjs`。需要實際修正時再使用 `node scripts/ensure-utf8.mjs --write`。 |
| 前端驗證 | 若改到 React / Vite 功能，優先跑既有 lint、build 或專案指定測試。 |
| 譜面格式驗證 | 若改到譜面匯入或 parser，優先跑 `scripts/` 中既有 normalize / stress / smoke 類腳本。 |
| 長期規格變更 | 同步更新 `樂理底層邏輯/README.md` 與 `專案記憶/` 中對應記憶。 |

## 鋼琴邏輯使用原則

| 任務 | 預設版本 |
|---|---|
| 匯入或修復舊 `.legacy.bak.txt` / legacy beat 譜 | 1 版 |
| 新增新歌、新 parser、新轉調、新播放顯示 | 2 版 |
| 討論未來演奏引擎、智慧伴奏、表情控制 | 3 版草案，但需先由使用者定義 |
| 播放器與譜面顯示層 | canonical event，不直接依賴原始文字格式 |

## 交接摘要格式

之後需要追加專案記憶時，使用這個格式：

```md
## YYYY-MM-DD

- 背景：
- 修改：
- 驗證：
- 後續：
```
