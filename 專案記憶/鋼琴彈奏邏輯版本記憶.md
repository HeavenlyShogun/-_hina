更新日期：2026-05-08

## 1 版：舊版鍵位譜邏輯

舊版邏輯以鍵盤字母直接表示音高，不先轉成樂理級數。它重視「原本手感」與「文字排版中的時間感」。

| 規則 | 記憶定義 |
|---|---|
| 節奏核心 | 每個 `/` 結束一個時間槽。時間槽長度由 `charResolution` 決定，預設可視為 16 分音符格線。 |
| 槽內分配 | 槽內的音符、和弦、休止與空白都會參與時間分配。空白不能被任意 trim，因為它可能代表等待。 |
| 單音 | 例如 `A`、`Q`、`Z`，直接對應鍵盤琴鍵。 |
| 快速連彈 | 例如 `AB`，表示同一 `/` 時間槽內連續觸發 A、B，依槽內事件數平均切 onset。 |
| 和弦 / 撥奏 | 例如 `(ABC)`，括號內視為同一事件，播放時可保留微小撥奏延遲。 |
| 休止 | `0` 是明確休止事件，輸出時仍保留給顯示與練習判定。 |
| 音域鍵位 | `A-J` 中音排、`Q-U` 高音排、`Z-M` 低音排，大小寫正規化為同一鍵位。 |
| 風險 | 適合保留舊譜，但不適合作為長期主資料模型；播放、顯示、轉調與驗證都需要額外相容層。 |

## 2 版：新版 canonical / 簡譜時值邏輯

新版邏輯是目前主線。所有輸入格式最後都必須轉成 canonical event，播放器與譜面顯示層只依賴正規化後的事件資料。

| 規則 | 記憶定義 |
|---|---|
| 時間基準 | `PPQ = 96`，一個四分音符等於 96 ticks。 |
| 內部時值 | 新版文字譜以一拍 `1.0` 為單位，token 先轉 `durationBeats`，再轉 `durationTicks = durationBeats * PPQ`。 |
| 小節驗證 | 例如 4/4 每小節必須等於 `4.0` 拍；遇到 `|` 時應檢查，不應默默補齊或略過錯誤。 |
| 音高 | 簡譜 `1 2 3 4 5 6 7` 表示級數，`0` 表示休止。 |
| 調性 | `globalKeyOffset` 表示主音相對 C 的半音偏移，`scaleMode` 表示 major / minor。 |
| 升降與八度 | `#` / `b` 處理臨時記號，`'` / `,` 或 `+` / `-` 處理八度位移。 |
| 多聲部 | 左右手或多聲部用不同 `trackId` 保存，各 track 分別驗證拍數，播放前合併 canonical event stream。 |
| canonical 欄位 | 核心欄位包含 `tick`、`durationTicks`、`time`、`durationSec`、`k`、`v`、`isRest`、`frequency`、`noteName`、`trackId`。 |
| 目標 | 讓匯入、顯示、播放、轉調、練習判定與測試都使用同一份事件合約。 |

## 長期記憶規則

- 說「1 版」時，指舊版鍵位譜 slash cell 邏輯。
- 說「2 版」時，指新版 canonical event / 簡譜時值邏輯。
- 說「3 版」時，除非使用者明確定義，先視為預留版本，不自行假設。
- 新功能優先接到 2 版主線；只有匯入舊譜或修相容性時才回到 1 版。
- 不論來源格式為何，播放與顯示層最終應吃 canonical event。
